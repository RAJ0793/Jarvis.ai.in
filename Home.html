üìò Installation & Setup Guides

Step-by-step instructions to get JARVIS up and running on your system

Minimum Requirements:

  • Processor: Intel i3 or equivalent (1 core capable)
  • RAM: 2GB minimum (4GB recommended)
  • Graphics: Intel Iris Xe Graphics or equivalent
  • Storage: 20GB dedicated space on J: drive
  • Python: Exactly version 3.10.0 (required)
  • OS: Windows 10/11, Linux, or macOS

‚ö†Ô∏è Important: JARVIS requires exactly Python 3.10.0. Other versions may cause compatibility issues.

Download Python 3.10.0:

  1. Visit the official Python website: python.org/downloads/release/python-3100/
  2. Scroll down to "Files" section
  3. Download the installer for your OS:
    • Windows: Windows installer (64-bit)
    • macOS: macOS 64-bit installer
    • Linux: Source tarball or use package manager

Installation Steps (Windows):

  1. Run the downloaded installer
  2. ‚úÖ IMPORTANT: Check "Add Python 3.10 to PATH"
  3. Click "Install Now"
  4. Wait for installation to complete
  5. Verify installation by opening Command Prompt and typing: python --version

üí° Verification: You should see "Python 3.10.0" in the output. If not, revisit PATH settings.

Why J: Drive?

JARVIS reserves 20GB on J: drive to ensure system stability and easy management. This prevents corruption of your main system drive and allows for clean uninstallation.

Creating J: Drive on Windows:

  1. Open Disk Management (Right-click Start ‚Üí Disk Management)
  2. Right-click on a drive with at least 20GB free space
  3. Select "Shrink Volume"
  4. Enter 20480 MB (20GB) in the shrink size
  5. Click "Shrink"
  6. Right-click the new unallocated space ‚Üí "New Simple Volume"
  7. Assign drive letter J:
  8. Format as NTFS
  9. Complete the wizard

‚ö†Ô∏è Alternative: If you cannot create a new partition, you can use an existing drive with at least 20GB free space, but ensure JARVIS files are in a dedicated folder.

Clone from GitHub:

  1. Open Command Prompt or Terminal
  2. Navigate to J: drive: cd J:\
  3. Clone the repository: git clone https://github.com/RAJ0793/Jarvis.ai.in.git
  4. Navigate to the folder: cd Jarvis.ai.in

Install Dependencies:

  1. Ensure you're in the JARVIS directory
  2. Install required packages: pip install -r requirements.txt
  3. Wait for all packages to download and install

‚úÖ Success: If installation completes without errors, you're ready to run JARVIS!

First Launch:

  1. Open Command Prompt or Terminal
  2. Navigate to JARVIS directory: cd J:\Jarvis.ai.in
  3. Run the main script: python main.py
  4. Wait for initialization (first run may take longer)
  5. Follow on-screen prompts for initial setup

Voice Commands:

Once JARVIS is running, you can use voice commands like:

  • "Hey JARVIS, what time is it?"
  • "JARVIS, open browser"
  • "JARVIS, search for Python tutorials"
  • "JARVIS, system status"

üí° Tip: Speak clearly and wait for the listening indicator before giving commands.

Configuring JARVIS:

Edit the config.json file to customize JARVIS behavior:

  • Voice Settings: Adjust speed, volume, and voice type
  • Wake Word: Change default wake word from "JARVIS"
  • Automation Rules: Set up custom automation triggers
  • API Keys: Add your API keys for web search and other services

Example config.json:

{ "wake_word": "JARVIS", "voice_speed": 150, "voice_volume": 0.8, "offline_mode": true, "auto_start": false }

Basic Operations:

1. Voice Commands

Activate JARVIS with the wake word, then speak your command clearly. Wait for acknowledgment before proceeding.

2. File Management

  • Ask JARVIS to open, create, or organize files
  • Use Smart Sorter to automatically categorize downloads
  • Enable Auto Renamer for batch file operations

3. Productivity Tools

  • Use Focus Shield to block distracting websites

üîß Troubleshooting Guide

Solutions to common issues and error messages

Problem: Python installer won't run or fails

Solutions:

  1. Download from Official Source: Only download from python.org
  2. Run as Administrator: Right-click installer ‚Üí "Run as administrator"
  3. Disable Antivirus Temporarily: Some antivirus software blocks Python installation
  4. Clear Temp Files: Delete files in C:\Windows\Temp
  5. Check Disk Space: Ensure you have at least 500MB free on C: drive

‚ö†Ô∏è Still not working? Try the Windows Store version of Python 3.10, but verify it's exactly version 3.10.0.

Problem: System shows different Python version

Check Current Version:

python --version

Solutions:

  1. Uninstall Other Versions:
    • Go to Control Panel ‚Üí Programs ‚Üí Uninstall a Program
    • Remove all Python versions except 3.10.0
  2. Fix PATH Variable:
    • Search "Environment Variables" in Windows
    • Edit "Path" under System Variables
    • Move Python 3.10 entries to the top
    • Remove paths to other Python versions
  3. Use py launcher: Run py -3.10 --version to verify

Problem: Cannot create J: drive or insufficient space

Solutions:

  1. Use Different Drive Letter:

    Edit JARVIS config to use a different drive (e.g., K: or L:)

  2. Free Up Space:
    • Delete temporary files
    • Uninstall unused programs
    • Move large files to external storage
  3. Use Existing Drive:

    Create a folder JARVIS on any drive with 20GB free space and update the config file

  4. External USB Drive:

    Assign letter J: to a USB drive with at least 20GB capacity

üí° Note: The J: drive requirement ensures JARVIS doesn't interfere with system files. Any dedicated 20GB space works.

Problem: "pip install -r requirements.txt" fails

Common Errors & Fixes:

1. "pip is not recognized"
  • Reinstall Python with "Add to PATH" checked
  • Or use: python -m pip install -r requirements.txt
2. "Could not find a version that satisfies the requirement"
  • Update pip: python -m pip install --upgrade pip
  • Check internet connection
  • Try installing packages individually
3. "Permission denied" errors
  • Run Command Prompt as Administrator
  • Or add --user flag: pip install -r requirements.txt --user
4. Timeout / Network errors
  • Increase timeout: pip install -r requirements.txt --timeout 100
  • Use different mirror: pip install -r requirements.txt -i https://pypi.org/simple

Problem: JARVIS doesn't respond to voice commands

Checklist:

  1. Microphone Settings:
    • Check Windows Sound Settings
    • Ensure microphone is not muted
    • Set correct default input device
    • Test microphone in Voice Recorder app
  2. Permissions:
    • Go to Settings ‚Üí Privacy ‚Üí Microphone
    • Enable "Allow apps to access your microphone"
    • Enable for Python specifically
  3. Audio Drivers:
    • Update audio drivers from Device Manager
    • Restart system after driver update
  4. PyAudio Installation: pip install pyaudio

    If this fails, download wheel file from here

Problem: Performance issues on low-end hardware

Optimization Steps:

  1. Enable Lite Mode:

    Edit config.json and set "lite_mode": true

  2. Disable Unused Features:
    • Turn off real-time monitoring
    • Disable background animations
    • Reduce voice response speed
  3. Close Background Apps:

    Free up RAM by closing unnecessary programs

  4. Increase Virtual Memory:
    • System Properties ‚Üí Advanced ‚Üí Performance Settings
    • Virtual Memory ‚Üí Change
    • Set custom size (Initial: 2048 MB, Maximum: 4096 MB)
  5. Run Single-Core Mode:

    In config, set "cpu_cores": 1 for stability

Problem: ImportError or ModuleNotFoundError

Solutions:

  1. Verify Virtual Environment:

    If using venv, ensure it's activated before running JARVIS

    J:\Jarvis.ai.in\venv\Scripts\activate
  2. Reinstall Specific Module: pip install [module_name]
  3. Check Python Path: python -c "import sys; print(sys.path)"

    Ensure JARVIS directory is in the path

  4. Reinstall All Dependencies: pip uninstall -r requirements.txt -y pip install -r requirements.txt

Problem: Application crashes immediately after starting

Debugging Steps:

  1. Check Error Logs:

    Open J:\Jarvis.ai.in\logs\error.log to see detailed error messages

  2. Run in Debug Mode: python main.py --debug
  3. Reset Configuration:
    • Rename config.json to config.json.old
    • JARVIS will create a new default config on next run
  4. Clean Installation:
    • Backup your config and data
    • Delete JARVIS folder
    • Re-clone from GitHub
    • Reinstall dependencies
  5. System Compatibility:

    Ensure your OS meets minimum requirements (Windows 10+, updated Linux kernel, macOS 10.14+)

JARVIS uses local speech recognition libraries that don't require internet:

Technology Stack:

  • SpeechRecognition: Captures and processes audio from your microphone
  • Pocketsphinx: Offline speech-to-text engine (included in dependencies)
  • pyttsx3: Text-to-speech for JARVIS responses

How it works:

  1. You speak the wake word ("JARVIS" by default)
  2. JARVIS activates and starts listening
  3. Your voice is captured and processed locally
  4. Command is recognized using offline models
  5. JARVIS executes the command and responds

Note: Offline recognition has ~85% accuracy. For better accuracy, you can enable online mode which uses Google's speech API.

‚öôÔ∏è Automation & Workflows

Automate repetitive tasks and create powerful workflows with JARVIS

Automation allows you to create intelligent workflows that execute tasks automatically based on triggers, schedules, or conditions. This saves time and reduces manual effort for repetitive operations.

Types of Automation:

  • Scheduled Tasks: Run actions at specific times (e.g., daily backup at 2 AM)
  • Event-Triggered: Execute when something happens (e.g., download completes ‚Üí organize files)
  • Conditional Logic: If-then rules based on system state
  • Voice-Activated: Custom voice commands to trigger complex workflows
  • Macro Recording: Record your actions and replay them

Example: Auto-Organize Downloads Folder

Step 1: Create Automation File

Navigate to J:\Jarvis.ai.in\automations\ and create organize_downloads.json

Step 2: Define the Automation

{ "name": "Organize Downloads", "trigger": "file_added", "watch_folder": "C:\\Users\\YourName\\Downloads", "actions": [ { "type": "move_file", "conditions": { "extension": [".jpg", ".png", ".gif"] }, "destination": "C:\\Users\\YourName\\Pictures" }, { "type": "move_file", "conditions": { "extension": [".pdf", ".docx"] }, "destination": "C:\\Users\\YourName\\Documents" } ] }

Step 3: Enable Automation

  1. Open JARVIS config: J:\Jarvis.ai.in\config.json
  2. Add to "enabled_automations": "organize_downloads"
  3. Restart JARVIS

‚úÖ Done! Now whenever you download a file, JARVIS automatically sorts it to the correct folder.

Example: Daily System Cleanup

{ "name": "Daily Cleanup", "trigger": "schedule", "schedule": "daily", "time": "03:00", "actions": [ { "type": "clear_cache", "folders": [ "C:\\Windows\\Temp", "C:\\Users\\YourName\\AppData\\Local\\Temp" ] }, { "type": "delete_old_files", "folder": "C:\\Users\\YourName\\Downloads", "older_than_days": 30 }, { "type": "notify", "message": "System cleanup completed!" } ] }

Schedule Options:

  • "schedule": "daily" - Every day at specified time
  • "schedule": "weekly" - Every Monday (or specify day)
  • "schedule": "hourly" - Every hour
  • "schedule": "interval" - Custom interval in minutes
